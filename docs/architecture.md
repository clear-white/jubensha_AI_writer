## 设计原则

### 1. Agent 独立性
每个 Agent 职责单一、独立运行，通过文件系统通信。Agent A 的输出文件就是 Agent B 的输入文件。

### 2. 确定性编排
工作流编排使用确定性脚本（Workflow），而非模型驱动。先做什么、再做什么、什么可以并行，都由编排逻辑决定。

### 3. 渐进式构建
创作过程分阶段进行，每个阶段产出可验证的中间产物。前一阶段完成并通过检查后，才能进入下一阶段。

### 4. 多层次质量控制
- **自动检查**：连续性引擎自动检测逻辑问题
- **专家审查**：Reviewer Agent 多维度评估
- **玩家测试**：真实玩家测试反馈

### 5. 一致性为本
剧本杀的推理质量高度依赖内部一致性。连续性引擎作为"编译器"，在每次变更后自动检测矛盾。

## 数据流

```
用户输入/Slash命令
    │
    ▼
Skill 处理入口
    │
    ▼
Workflow 编排 (确定多个Agent的执行顺序)
    │
    ├─→ Agent A (并行)
    ├─→ Agent B (并行)  ← 无依赖关系的Agent并行执行
    │
    ▼
    Agent C (串行，依赖 A 和 B 的输出)
    │
    ▼
    连续性引擎验证
    │
    ▼
    用户确认/反馈
    │
    ▼
    下一阶段
```

## 文件组织

### 创作中剧本结构
```
output/<剧本名称>/
├── info.md                    # 基础信息
├── world/                     # 世界观
├── synopsis.md                # 剧情大纲
├── timeline.md                # 时间线
├── characters/                # 角色档案
├── clues/                     # 线索系统
├── scripts/                   # 剧本正文
├── props/                     # 道具物料
├── dm-guide/                  # DM指南
└── review/                    # 审核记录
```

### 输出成品结构
```
output/<剧本名称>/
├── 剧本正文.md                # 主持人用完整剧本
├── 角色剧本-角色A.md           # 玩家角色剧本
├── 角色剧本-角色B.md
├── ...
├── DM指南.md                  # 主持人手册
├── 线索卡.pdf                 # 可打印线索卡
├── 物证卡片/                  # 物证图片/卡片
├── 游戏规则说明.md             # 规则讲解
└── 物料清单.md                # 开本前准备清单
```
