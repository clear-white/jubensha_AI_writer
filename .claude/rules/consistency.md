# 一致性检查规则

## 检查维度

### 1. 时间线一致性
- 所有角色的时间线必须能够共存于同一时空中
- 同一时间点，同一角色不能出现在两个地点
- 任何两个角色的时间线不能有不可调和的矛盾
- 关键事件的时间必须精确到分钟级别

### 2. 因果关系一致性
- 每个事件必须有明确的起因
- 事件链不能有断裂或跳跃
- 关键转折点必须满足前置条件
- 所有"契诃夫之枪"必须在结局前被使用

### 3. 角色一致性
- 角色的行为必须符合其性格设定
- 角色的知识状态必须在不同阶段保持一致
- 角色的动机不能随意变更（除非有合理的剧情原因）
- 角色的言语和行动不能前后矛盾

### 4. 信息一致性
- 角色A知道的信息，如果通过合理途径可以被角色B知道，则必须处理这种可能性
- 公开场合发生的事件，所有在场角色都应有该信息
- 秘密信息不能通过不合理的方式泄露
- 线索之间的关联必须逻辑自洽

### 5. 世界规则一致性
- 不能违反已设定的世界观规则
- 特殊能力/技能的使用必须符合限制条件
- 物理规则必须前后一致（除非设定中明确说明）
- 社会规则和文化习俗不能随意改变

## 检查方法

### 自动检查（连续性引擎）
```json
{
  "timeline_check": {
    "conflicts": [],
    "gaps": [],
    "status": "passed/failed"
  },
  "causality_check": {
    "broken_chains": [],
    "unresolved_foreshadowing": [],
    "status": "passed/failed"
  },
  "character_check": {
    "inconsistencies": [],
    "knowledge_conflicts": [],
    "status": "passed/failed"
  }
}
```

### 人工复查
- 通读剧本，模拟游戏流程
- 站在每个角色的视角检查信息流
- 检查每个隐藏秘密是否可能被提前发现
- 检查不同结局的触发条件是否都合理

## 严重等级

| 等级 | 定义 | 处理方式 |
|------|------|---------|
| 🔴 致命 | 逻辑矛盾导致剧本无法进行 | 必须修复 |
| 🟡 严重 | 明显的不一致，影响体验 | 推荐修复 |
| 🔵 轻微 | 小细节不一致 | 选择性修复 |
| 🟢 通过 | 完全一致 | 保持 |
