# 创作工作流规则

## 创作阶段与节点

### Phase 1: 创意与构思
**检查点 1.1** — 创意评估
- [ ] 核心创意是否新颖有趣
- [ ] 目标玩家群体是否明确
- [ ] 类型定位是否清晰

**检查点 1.2** — 可行性评估
- [ ] 核心诡计是否可执行
- [ ] 世界观设定是否自洽
- [ ] 是否需要特殊物料

### Phase 2: 设计与架构
**检查点 2.1** — 剧情完整性
- [ ] 剧情大纲完整（起承转合）
- [ ] 核心诡计明确且有逻辑
- [ ] 关键反转点已确定

**检查点 2.2** — 角色体系完整性
- [ ] 角色数量符合要求
- [ ] 每个角色有清晰定位
- [ ] 关系网络已构建

**检查点 2.3** — 时间线一致性
- [ ] 绝对时间线无冲突
- [ ] 所有角色时间线兼容
- [ ] 关键事件时间精确

### Phase 3: 详细创作
**检查点 3.1** — 线索系统完整性
- [ ] 线索数量符合难度
- [ ] 证据链完整无断裂
- [ ] 推理路径多样

**检查点 3.2** — 剧本正文完整性
- [ ] 分幕结构完整
- [ ] 角色剧本完整
- [ ] 对白和行动指令清晰

**检查点 3.3** — 道具物料完整性
- [ ] 物证卡片齐全
- [ ] 场景布置说明完整
- [ ] 配套物料齐全

### Phase 4: 审核与优化
**检查点 4.1** — 全量审核
- [ ] 连续性引擎无报错
- [ ] 多维度审核通过
- [ ] 所有严重问题已修复

**检查点 4.2** — 测试
- [ ] 内部模拟测试
- [ ] 真实玩家测试
- [ ] 测试反馈已整合

### Phase 5: 成品发布
**检查点 5.1** — 最终检查
- [ ] DM指南完整
- [ ] 输出格式正确
- [ ] 所有文件整理完毕

## Agent 调用规范

### 并行调用规则
- 无依赖关系的任务始终并行
- 同一阶段的独立 Agent 并行
- 读取操作不影响写入，可以并行

### 串行依赖规则
- 有明确输入输出依赖的必须串行
- 世界观 → 剧情 → 角色的依赖链
- 审核必须在所有创作完成后

### Agent 之间通信
- 通过文件系统共享信息（而非消息传递）
- 后序 Agent 读取前序 Agent 的输出文件
- 使用标准化的文件命名和格式

## 错误处理

### Agent 调用失败
1. 重试 1 次
2. 如果仍失败，记录错误并继续
3. 在审核阶段重点检查该部分

### 一致性检查失败
1. 严重错误：停止流程，修复后再继续
2. 轻微警告：记录问题，继续流程，在审核阶段统一修复

### 用户中途修改需求
1. 记录修改内容到 memory
2. 评估影响范围
3. 重新触发受影响的子流程
