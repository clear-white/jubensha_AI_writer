# 记忆系统规则

## 概述
使用 Claude Code 的持久化记忆系统 (`memory/` 目录) 来跨会话保持项目状态和创作上下文。

## 什么需要记忆

### 用户偏好
- 偏好的剧本类型
- 偏好的创作风格和方向
- 对之前作品的评价和反馈
- 用户的专业程度（玩家/DM/作者）

### 项目状态
- 当前正在创作的剧本及进度
- 已完成的剧本列表
- 关键的创作决策和理由
- 被否决的创意方向和原因

### 创作经验
- 之前作品中遇到的问题和解决方案
- 用户提出的特殊要求
- 创作流程中需要改进的地方

## 记忆格式

```markdown
---
name: <短横线命名>
description: <一行摘要>
metadata:
  type: user | project | feedback | reference
---

<记忆内容>

**关联记忆:** [[related-memory-1]], [[related-memory-2]]
```

## 记忆类型

- **user**: 用户信息（角色、偏好、习惯）
- **project**: 项目状态和决策记录
- **feedback**: 用户的反馈和修正指导
- **reference**: 外部参考资料和灵感来源

## 记忆使用规则

1. 开始新剧本项目时，读取所有相关记忆
2. 完成重要创作决策时，保存到记忆
3. 收到用户反馈时，立即保存
4. 每次会话开始时，加载记忆索引 (MEMORY.md)
5. 定期检查和合并相关记忆
