# Pre-Tool Hook

## 用途
在执行工具调用前执行的验证逻辑。

## 检查清单

### 文件写入前检查
- [ ] 写入路径是否在项目目录内
- [ ] 是否覆盖已有文件（如果是，检查是否是预期操作）
- [ ] 写入内容是否包含敏感信息（API密钥、密码等）

### Agent 调用前检查
- [ ] 是否已在 memory 中记录了当前项目状态
- [ ] 是否有足够的上下文信息传递给 Agent
- [ ] Agent 之间是否存在循环依赖

### 工作流执行前检查
- [ ] 所需的前置文件是否存在
- [ ] 依赖链是否完整
- [ ] 是否在正确的创作阶段

## 自动行为

1. 如果检测到 `output/` 路径下的写入，自动创建对应子目录
2. 如果在没有初始化项目时执行创作命令，提示先运行 `/init-script`
3. 执行 `/create-script` 前，检查是否存在同名输出目录，提示确认覆盖
